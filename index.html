Какие модели должны быть в NewsPaper?

  1. Модель Author, содержащая объекты всех авторов.
  2. Модель Category - категории новостей/статей — темы, которые они отражают (бизнес, технологии, образование, образ жизни и т.д.).
  3. Модель Post - статьи и новости, которые создают пользователи. Каждый объект может иметь одну или несколько категорий.
  4. Модель PostCategory - промежуточная модель для связи «многие ко многим».
  5. Модель Comment - хранение комментариев, оставляемых под каждой новостью/статьёй.

Поля моделей

1) Поля модели Author:
1.1) cвязь «один к одному» с встроенной моделью пользователей User;
1.2) рейтинг пользователя (ниже будет дано описание того, как этот рейтинг можно посчитать).

class Author(models.Model):
    authorUser = models.OneToOneField(User, on_delete=models.CASCADE)
    authorRating = models.SmallIntegerField(default=0)

2) Поля модели Category:
2.1) название категории - поле должно быть уникальным (в определении поля необходимо написать параметр unique = True).

class Category(models.Model):
    categoryName = models.CharField(max_length=64, unique=True)

3) Поля модели Post:
3.1) связь «один ко многим» с моделью Author;
3.2) поле с выбором — «статья» или «новость»;
3.3) автоматически добавляемая дата и время создания;
3.4) связь «многие ко многим» с моделью Category (с дополнительной моделью PostCategory);
3.5) заголовок статьи/новости;
3.6) текст статьи/новости;
3.7) рейтинг статьи/новости.

class Post(models.Model):
    article = 'A'
    news = 'N'

    POST_TYPE = [
        (article, "Статья"),
        (news, "Новость")
    ]

    postAuthor = models.ForeignKey(Author, on_delete=models.CASCADE)
    post_type = models.CharField(max_length=1, choices=POST_TYPE, default=article)
    postCreated = models.DateTimeField(auto_now_add=True)
    postCats = models.ManyToManyField(Category, through='PostCategory')
    postTitle = models.CharField(max_length=128)
    postText = models.TextField()
    postRating = models.SmallIntegerField(default=0)

4) Поля модели PostCategory:
4.1) связь «один ко многим» с моделью Post;
4.2) связь «один ко многим» с моделью Category.

class PostCategory(models.Model):
    fromPost = models.ForeignKey(Post, on_delete=models.CASCADE)
    fromCategory = models.ForeignKey(Category, on_delete=models.CASCADE)

5) Поля модели Comment:
5.1) связь «один ко многим» с моделью Post;
5.2) связь «один ко многим» со встроенной моделью User (комментарии может оставить любой пользователь, необязательно автор);
5.3) текст комментария;
5.4) дата и время создания комментария;
5.5) рейтинг комментария.

class Comment(models.Model):
    commentForPost = models.ForeignKey(Post, on_delete=models.CASCADE)
    commentUser = models.ForeignKey(User, on_delete=models.CASCADE)
    commentText = models.TextField()
    commentCreated = models.DateTimeField(auto_now_add=True)
    commentRating = models.SmallIntegerField(default=0)

Методы и свойства моделей

Описанные выше пять моделей должны реализовать следующие методы:

1) Методы like() и dislike() в моделях 5)Comment и 3)Post, которые увеличивают/уменьшают рейтинг на единицу.

Рейтинг для postRating:

    def like(self):

    def like(self):

2) Метод preview() модели 3)Post, который возвращает начало статьи (предварительный просмотр) длиной 124 символа и добавляет многоточие в конце.

Примечание: в Python индексы начинаются с 0, поэтому первый элемент имеет индекс 0, второй - индекс 1 и т.д.
Срез определяется с помощью квадратных скобок [ ], берутся все элементы из последовательности начиная с указанного и до указанного индекса (не включая его).

1-ое решение используем конкатенацию (объединение) строк с помощью оператора "+":

    def preview(self):
        return self.postText[:124] + '...'

2-ое решение используем форматирование строки с помощью f-строки:

    def preview(self):
        return f"{self.postText[:124]}..."

3-е решение используем условное выражение для определения длины предварительного просмотра.
Если длина текста postText превышает 124 символа, то preview_length будет равно 124, иначе оно будет равно длине postText.
Затем возвращается начало текста postText длиной preview_length, с добавлением многоточия в конце.

    def preview(self):
        preview_length = 124 if len(self.postText) > 124 else len(self.postText)
        return self.postText[:preview_length] + '...'

4-е решение используем условное выражение для проверки длины текста.
Если длина текста postText превышает 124 символа, то возвращается начало текста postText длиной 124 символа, с добавлением многоточия в конце.
В противном случае, возвращается весь текст postText без изменений.

    def preview(self):
        return self.postText[:124] + '...' if len(self.postText) > 124 else self.postText

5-е решение опять же используем условное выражение для проверки длины текста.
Значение длины текста устанавливаем в переменной, которая определяет максимальную длину предварительного просмотра.
Если длина текста postText превышает длину предварительного просмотра,
то postText урезается до длины предварительного просмотра и добавляется в конце многоточие.
В ином случае, возвращается весь текст postText без изменений.

    def preview(self):
        preview_length = 124
        return self.postText[:preview_length] + '...' if len(self.postText) > preview_length else self.postText

6-е решение используем условный оператор if-else для проверки длины текста self.postText.
Если длина текста превышает preview_length, то возвращается начало текста длиной preview_length с добавлением многоточия в конце.
В противном случае, возвращается весь текст postText без изменений.

    def preview(self):
        preview_length = 124
        if len(self.postText) > preview_length:
            return self.postText[:preview_length] + '...'
        else:
            return self.postText

7-е решение используем условный оператор if-else для проверки длины текста self.postText.
Если длина текста меньше или равна preview_length, возвращается весь текст статьи.
В ином случае, возвращается начало текста указанной длины и добавляется многоточие в конце.

    def preview(self):
        preview_length = 124
        ellipsis = '...'
        if len(self.postText) <= preview_length:
            return self.postText
        else:
            return self.postText[:preview_length] + ellipsis

Среди этих семи решений наиболее элегантными и лаконичными являются решения 5 и 2.
Решение 5 - это сочетание решений 3 и 4, оно урезает строку только при необходимости.
Оно удобно, поскольку указание длины предварительного просмотра использовано один раз, и элегантно, т.к. объединяет логику усечения и многоточия с использованием тернарного (условного) оператора, задающего условие в одной строке кода.
Решение 2 самое лаконичное - оно использует f-строку для достижения желаемого результата в одной строке кода.

Исходя из этого, берем вариант 2 и немного модифицируем его.
Поскольку в дальнейшем при выполнении задания мы будем добавлять в базу данных текст, отформатированный с помощью тегов HTML, нам нужно чтобы в предварительный просмотр выводился текст без HTML-тегов.
Для этого можно использовать функцию strip_tags из модуля django.utils.html.
Эта функция удаляет все HTML-теги из строки.
Изменим код варианта 2, чтобы использовать эту функцию:

from django.utils.html import strip_tags # для удаления всех HTML-тегов из строки - эта строка добавляется в начало файла models.py перед строкой # Create your models here.

Модифицированный вариант решения 2 с функцией strip_tags из модуля django.utils.html.
Применяем функцию strip_tags к self.postText, чтобы удалить все HTML-теги из текста:

    def preview(self):
            text_without_tags = strip_tags(self.postText)[:124]
            return f"{text_without_tags}..."

3) Метод update_rating() модели 1)Author, который обновляет рейтинг пользователя, переданный в аргумент этого метода.
Он состоит из следующего:
3.1) суммарный рейтинг каждой статьи автора умножается на 3;
3.2) суммарный рейтинг всех комментариев автора;
3.3) суммарный рейтинг всех комментариев к статьям автора.

    def update_rating(self):

# 1) суммарный рейтинг каждой статьи postRating автора postAuthor умножается на 3
# с помощью post_set.all() - получим все связанные с автором посты в модели Post, 
# и с помощью aggregate получаем все значения поля рейтинг и суммируем их
        sumPR = self.post_set.all().aggregate(sumPostRating=Sum('postRating'))
# создадим промежуточную переменную
        pRate = 0
        pRate += sumPR.get('sumPostRating')

# 2) суммарный рейтинг всех комментариев commentRating автора authorUser
        sumAuthorCR = self.authorUser.comment_set.all().aggregate(sumAuthorComRating=Sum('commentRating'))
        authorCRate = 0
        authorCRate += sumAuthorCR.get('sumAuthorComRating')

# 3) суммарный рейтинг всех комментариев commentRating к статьям автора postAuthor
        sumPostCR = Comment.objects.filter(commentForPost__postAuthor=self).values('commentRating').aggregate(sumPostComRating=Sum('commentRating'))
        postCRate = 0
        postCRate += sumPostCR.get('sumPostComRating')

# print('pRate =', pRate * 3, 'authorCRate =', authorCRate, 'postCRate =', postCRate)

        self.authorRating = pRate * 3 + authorCRate + postCRate
        self.save()

Важно: Если в файл models.py внесены корректировки после применения миграций, то следует опять выполнить команды:

python manage.py migrate

Примечание: При выполнении команды python manage.py makemigrations в Django, существующие строки в затронутых таблицах базы данных не должны пропадать.
Команда makemigrations создает файлы миграции, которые описывают изменения в моделях Django. Эти файлы миграции содержат инструкции о том, как изменить схему базы данных, чтобы соответствовать изменениям в моделях.
Выполнение миграций может привести к изменению структуры базы данных и, в некоторых случаях, может потребоваться удаление или изменение существующих данных. Например, если добавляется новое поле в модель и создается миграция, Django может потребовать заполнить это поле для существующих записей в таблице.
Поэтому, перед выполнением миграций, особенно если они вносят значительные изменения в схему базы данных, рекомендуется создать резервную копию базы данных.

4.1) Cоздание текстов статей/ новостей:
Примечание:
В поле postText можно вставлять обычный текст или текст, отформатированный с помощью тегов HTML, в зависимости от ваших потребностей.

Примечание: В базах данных нумерация обычно начинается с 1. Это означает, что первая запись в таблице будет иметь идентификатор (id) равный 1, вторая запись - id равный 2 и т.д. Однако, в некоторых случаях, можно настроить базу данных таким образом, чтобы нумерация начиналась с 0. Это зависит от конкретной системы управления базами данных (СУБД) и ее настроек. В Django, по умолчанию, нумерация идентификаторов в базе данных начинается с 1.

                article2,

                comment3,

# 50 рандомных лайков/ дислайков
